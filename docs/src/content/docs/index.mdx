tmakit is not a wrapper over an SDK. The value is in the layers _around_ it: a rich
dev-time client mock, opinionated runtime helpers the SDK deliberately leaves out, and
testing utilities. It works with both `@telegram-apps/*` and `@tma.js/*`, because they
share the same Telegram bridge protocol.

<CardGrid stagger>
	<Card title="Runtime kit" icon="rocket">

	<Card title="Devtools" icon="laptop">
